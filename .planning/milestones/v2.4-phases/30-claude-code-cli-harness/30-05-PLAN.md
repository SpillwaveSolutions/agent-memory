  <action>
In `crates/memory-ingest/src/main.rs`, modify the `rt.block_on` async block (around line 136-141) to read the `MEMORY_DAEMON_ADDR` environment variable before connecting.

Current code:
```rust
rt.block_on(async {
    if let Ok(mut client) = MemoryClient::connect_default().await {
        let _ = client.ingest(event).await;
    }
});
```

Replace with:
```rust
rt.block_on(async {
    let client_result = if let Ok(addr) = std::env::var("MEMORY_DAEMON_ADDR") {
        MemoryClient::connect(&addr).await
    } else {
        MemoryClient::connect_default().await
    };
    if let Ok(mut client) = client_result {
        let _ = client.ingest(event).await;
    }
});
```

This preserves fail-open behavior (if env var is unset, falls back to default port 50051). When MEMORY_DAEMON_ADDR is set (as common.bash's `ingest_event()` does), the binary connects to the specified address.
