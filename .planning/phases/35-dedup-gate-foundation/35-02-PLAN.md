Modify `crates/memory-service/src/novelty.rs`:

1. Add imports: `use memory_types::dedup::InFlightBuffer;` and `use std::sync::RwLock;` (std, not tokio -- the buffer is fast enough for sync lock).

2. Update `use memory_types::config::NoveltyConfig;` to `use memory_types::config::DedupConfig;`. Then add `use memory_types::config::DedupConfig as NoveltyConfig;` OR simply use DedupConfig throughout the file (since NoveltyConfig is a type alias, either works). Prefer using `DedupConfig` directly and updating all references in this file.

3. Create `InFlightBufferIndex` struct:
   ```rust
   pub struct InFlightBufferIndex {
       buffer: Arc<RwLock<InFlightBuffer>>,
   }

   impl InFlightBufferIndex {
       pub fn new(buffer: Arc<RwLock<InFlightBuffer>>) -> Self {
           Self { buffer }
       }

       pub fn buffer(&self) -> &Arc<RwLock<InFlightBuffer>> {
           &self.buffer
       }
   }
   ```

4. Implement `VectorIndexTrait` for `InFlightBufferIndex`:
   - `is_ready(&self) -> bool` -- always returns `true` (in-memory, always available)
   - `search(&self, embedding: &[f32], top_k: usize) -> Result<Vec<(String, f32)>, String>` -- acquire read lock on buffer, call `find_similar(embedding, 0.0)` to get best match (threshold filtering happens in NoveltyChecker, not here), return as `vec![(id, score)]` if found or `vec![]` if not. Use `top_k` parameter: if top_k==0 return empty. The `find_similar` threshold of 0.0 means "return any match" -- let the caller (NoveltyChecker) do threshold comparison.

   IMPORTANT: The existing `check_similarity` method on line 256 does `Ok(*score <= self.config.threshold)` where score comes from VectorIndexTrait.search(). This means: score <= threshold = novel (true), score > threshold = duplicate (false). The InFlightBuffer's `find_similar` returns cosine similarity (higher = more similar). So the VectorIndexTrait adapter must return the raw similarity score -- the existing threshold comparison in check_similarity handles the rest correctly.

5. Add an `in_flight_buffer` field to `NoveltyChecker`:
   ```rust
   in_flight_buffer: Option<Arc<RwLock<InFlightBuffer>>>,
   ```

6. Update `NoveltyChecker::new()` to accept an additional `in_flight_buffer: Option<Arc<RwLock<InFlightBuffer>>>` parameter. Store it in the struct.

7. Add a method `pub fn push_to_buffer(&self, event_id: &str, embedding: &[f32])` that pushes an embedding into the in_flight_buffer if present. This is called by the caller (ingest pipeline in Phase 36) AFTER should_store returns true. For Phase 35, the unit tests will call it directly.

   IMPORTANT: Do NOT modify `should_store()` to auto-push. The push must happen AFTER the event is actually stored (which is the ingest pipeline's responsibility in Phase 36). Adding auto-push would push embeddings for events that might fail to store.

8. Add a convenience constructor:
   ```rust
   pub fn with_in_flight_buffer(
       embedder: Option<Arc<dyn EmbedderTrait>>,
       buffer: Arc<RwLock<InFlightBuffer>>,
       config: DedupConfig,
   ) -> Self
   ```
   This creates the checker with `InFlightBufferIndex` wrapping the buffer as the vector_index, AND stores the buffer reference for push_to_buffer. This is the constructor Phase 36 will use.

9. Preserve ALL existing tests. Update their NoveltyConfig references to DedupConfig (the alias means both work, but prefer consistency).
