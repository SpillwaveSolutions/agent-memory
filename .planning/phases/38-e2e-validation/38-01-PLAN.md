Create `crates/e2e-tests/tests/dedup_test.rs` with the following tests. All tests use `#[tokio::test]` and `pretty_assertions`.

**Test 1: `test_dedup_duplicate_stored_but_not_indexed`**
- Create TestHarness, MockEmbedder with uniform_normalized(384), InFlightBuffer(256, 384)
- Create NoveltyChecker via `NoveltyChecker::with_in_flight_buffer(Some(embedder), buffer, DedupConfig { enabled: true, threshold: 0.85, min_text_length: 10, ..Default::default() })`
- Create MemoryServiceImpl via `MemoryServiceImpl::new(harness.storage.clone())`, call `service.set_novelty_checker(checker)`
- Ingest first event via `service.ingest_event(Request::new(IngestEventRequest { event: Some(create_proto_event(...)) }))` -- assert `created=true, deduplicated=false`
- Ingest second event (different event_id, same session, different text but SAME mock embedding) -- assert `created=true, deduplicated=true`
- Assert both events exist in storage: `harness.storage.get_event(&event_id).unwrap().is_some()`
- Assert storage stats show event_count = 2
- Verify outbox behavior: use `harness.storage.get_outbox_entries(0, 100)` and assert only 1 outbox entry exists (for the novel event, not the duplicate). The outbox_entries list should have length 1.

**Test 2: `test_dedup_novel_events_all_indexed`**
- Same setup but use TWO different MockEmbedders is not possible (NoveltyChecker takes one embedder). Instead, use a MockEmbedder that returns DIFFERENT embeddings per call. Create a `SequentialEmbedder` struct with `Arc<Mutex<Vec<Vec<f32>>>>` of embeddings to return in order. First call returns vec_a (e.g., unit vector along dim 0), second returns vec_b (unit vector along dim 1). These have cosine similarity ~0 so both are novel.
- Ingest two events -- both should have `deduplicated=false`
- Assert outbox has 2 entries

**Test 3: `test_dedup_structural_events_bypass_gate`**
- Setup with MockEmbedder returning uniform_normalized (all identical embeddings)
- Ingest a normal UserMessage event -- novel, stored + outbox
- Ingest a SessionStart structural event via `create_proto_event_structural()` -- should bypass dedup entirely (per DEDUP-04), `deduplicated=false`
- Assert structural event has outbox entry (outbox_entries count = 2)
- Assert `deduplicated=false` on the structural event response even though embeddings would match

**Test 4: `test_dedup_response_fields`**
- Same setup as Test 1
- Ingest novel event, verify response: `created=true, deduplicated=false`
- Ingest duplicate event, verify response: `created=true, deduplicated=true`
- Ingest structural event, verify response: `created=true, deduplicated=false`

Use imports: `use std::sync::{Arc, RwLock, Mutex}`, `use tonic::Request`, `use memory_service::{MemoryServiceImpl, NoveltyChecker, pb::{IngestEventRequest, memory_service_server::MemoryService}}`, `use memory_types::{config::DedupConfig, dedup::InFlightBuffer}`, `use e2e_tests::{TestHarness, MockEmbedder, uniform_normalized, create_proto_event, create_proto_event_structural}`, `use memory_service::novelty::EmbedderTrait`.

For `SequentialEmbedder` (Test 2), define it locally in the test file. It implements `EmbedderTrait`, pops from a VecDeque on each embed call.
